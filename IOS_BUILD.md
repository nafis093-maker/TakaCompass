# iOS build guide

Taka Compass now has an iOS target (Capacitor). The native iOS project lives in
`ios/`. Building/signing requires **macOS + Xcode** — it can't be done on Linux.

## What works on iOS
- Local notifications (bill/EMI reminders)
- Receipt photos (camera/library — usage strings are in `Info.plist`)
- Everything in the web app (transactions, wallets, Plan, Wrapped, etc.)

## What does NOT work on iOS
- **Automatic SMS capture / inbox scan** — Apple does not allow reading texts.
  The app detects the platform and hides those controls on iOS; use the SMS
  *paste* tool or manual entry instead. (This is Android-only by design.)

## Option A — Cloud build (no Mac needed to verify it compiles)
The `Build iOS app` GitHub Action builds an **unsigned simulator** app on a
macOS runner. Actions tab → "Build iOS app" → Run workflow → download the
`taka-compass-ios-simulator` artifact. Unzip and drag `App.app` onto a running
iOS Simulator (`xcrun simctl install booted App.app`). This proves the build and
is great for testing, but a simulator build can't run on a physical iPhone.

## Option B — Local build on your Mac (device / App Store)
Prerequisites: macOS, Xcode, CocoaPods (`sudo gem install cocoapods`), and an
Apple Developer account for device installs or TestFlight.

```bash
npm install
npm run build
npx cap sync ios          # copies web build + runs pod install
npx cap open ios          # opens Xcode
```
In Xcode:
1. Select the **App** target → **Signing & Capabilities** → pick your Team
   (Xcode will create a provisioning profile). The bundle id is
   `app.takacompass.money` — change it if it's taken.
2. Add the **Push/Notifications**? Not required; local notifications need no
   capability. Camera/photo permission strings are already in `Info.plist`.
3. Pick a device (or "Any iOS Device") and **Product → Run** (device) or
   **Product → Archive** → Distribute App → TestFlight/App Store.

## Signed CI (optional, advanced)
To make CI produce a signed `.ipa`, add repo secrets for your distribution
certificate (`.p12` + password) and provisioning profile, import them in the
workflow, then replace the simulator step with:
```bash
xcodebuild -workspace ios/App/App.xcworkspace -scheme App \
  -configuration Release -archivePath build/App.xcarchive archive
xcodebuild -exportArchive -archivePath build/App.xcarchive \
  -exportOptionsPlist ExportOptions.plist -exportPath build/ipa
```
Tools like `apple-actions/import-codesign-certs` simplify the secret import.

## Notes
- After changing web code or plugins, always run `npx cap sync ios` before
  building so `ios/App/App/public` and the Pods are up to date.
- `ios/App/Pods/` is generated by `pod install`; it's fine to leave it
  uncommitted (CI and local builds regenerate it).
